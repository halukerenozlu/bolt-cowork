# Code of Conduct

## Our Commitment

We want bolt-cowork to be a safe, respectful, and inclusive community for everyone.
Contributors, maintainers, and participants are expected to collaborate in good faith and treat each other with professionalism.

## Expected Behavior

Examples of behavior that helps build a positive environment:

- Use welcoming and respectful language.
- Assume good intent, ask clarifying questions, and give constructive feedback.
- Focus criticism on ideas and implementation, not people.
- Respect different backgrounds, skill levels, and communication styles.
- Accept responsibility, apologize when needed, and work to repair harm.

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, intimidation, threats, or discriminatory comments.
- Personal attacks, insults, trolling, or sustained disruption.
- Sexual language or imagery in project spaces.
- Publishing private information (doxxing) without explicit consent.
- Any conduct that a reasonable person would consider inappropriate in a professional open-source community.

## Scope

This Code of Conduct applies in all bolt-cowork project spaces, including:

- GitHub issues, pull requests, discussions, and reviews
- Documentation, commit messages, and comments
- Any public or private communication reasonably connected to project collaboration

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct.
They may remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that violate these standards.

Possible enforcement actions include:

- Warning and request for behavior change
- Temporary suspension from project spaces
- Permanent ban from participation in project spaces

Enforcement decisions aim to protect the community and will be applied fairly and consistently.

## Reporting

If you experience or witness unacceptable behavior, report it privately to the maintainer through GitHub private contact channels (for example, the maintainer contact information listed on the repository owner profile).

Please include relevant details such as:

- What happened
- Where and when it happened
- Links, screenshots, or other supporting context

All good-faith reports will be reviewed promptly and handled as confidentially as possible.
Retaliation against anyone who reports a concern in good faith is not allowed.
