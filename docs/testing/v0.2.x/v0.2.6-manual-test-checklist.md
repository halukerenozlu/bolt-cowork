# Bolt Cowork v0.2.6 — Manuel Test Checklist

> Her maddeyi terminalde dene, sonucu ✅ / ❌ olarak işaretle.
> Test ortamı: boş bir klasörde `bolt-cowork --dir ./test-workspace` ile başla.

---

## 1. CLI Başlangıç

- [ ] `bolt-cowork --help` → flagler ve açıklamalar görünüyor
- [ ] `bolt-cowork --version` → sürüm bilgisini config/LLM yüklemeden gösteriyor
- [ ] `bolt-cowork --dir ./test-workspace` → REPL açılıyor, banner (ASCII-only) görünüyor
- [ ] `bolt-cowork --dir /nonexistent` → anlamlı hata mesajı, crash yok
- [ ] `bolt-cowork --provider nonexistent` → provider bulunamadı hatası
- [ ] `bolt-cowork --approval invalid` → geçersiz approval mode hatası

## 2. Config Yükleme

- [ ] `~/.bolt-cowork/config.yaml` yokken başlat → varsayılan config ile çalışıyor
- [ ] `config.yaml` içinde env var kullan (`${API_KEY}`) → genişletiliyor
- [ ] Geçersiz YAML → açıklayıcı parse hatası, panic yok
- [ ] Tilde (`~`) path genişletmesi çalışıyor (ör. `skills_dir: ~/my-skills`)

## 3. REPL Temel Komutlar

- [ ] Boş input (sadece Enter) → hata yok, yeni prompt
- [ ] `/help` → komut listesi görünüyor
- [ ] `/dir` → mevcut çalışma dizinini gösteriyor
- [ ] `/dir ./subdir` → dizin değişiyor (subdir varsa)
- [ ] `/dir /outside/sandbox` → sandbox dışı, reddediliyor
- [ ] `/dir -` → önceki dizine dönüyor
- [ ] `/quit` veya Ctrl+C → temiz çıkış, "goodbye" mesajı
- [ ] Bilinmeyen komut (`/asdf`) → "unknown command" hatası

## 4. Agent Loop — Dosya İşlemleri

- [ ] "create a file called hello.txt with content 'merhaba'" → dosya oluşuyor, içerik doğru
- [ ] "read hello.txt" → içerik gösteriliyor
- [ ] "rename hello.txt to selam.txt" → isim değişiyor
- [ ] "copy selam.txt to backup.txt" → kopya oluşuyor
- [ ] "delete backup.txt" → onay isteniyor (dangerous), onaylayınca siliniyor
- [ ] "list files" → mevcut dosyalar listeleniyor
- [ ] "create directory subdir" → klasör oluşuyor

## 5. Approval Gate (Onay Kapısı)

### 5a. Full Approval Mode (`--approval full`)
- [ ] Her adımda onay isteniyor (plan + execute)
- [ ] Reject yapınca → işlem iptal, dosya değişmemiş
- [ ] ApproveAll seçince → kalan adımlar otomatik onaylanıyor

### 5b. Dangerous-Only Mode (`--approval dangerous-only`)
- [ ] read/list gibi güvenli işlemler → otomatik onay, soru yok
- [ ] delete/overwrite → onay isteniyor, `[DANGEROUS]` etiketi + neden görünüyor
- [ ] ApproveAll seçtikten sonra başka bir dangerous adım gelince → YİNE onay istiyor
- [ ] Non-dangerous adımlar ApproveAll sonrası → otomatik geçiyor

### 5c. None Mode (`--approval none`)
- [ ] Hiçbir adımda onay istemiyor
- [ ] Dangerous işlemler bile direkt çalışıyor (uyarı: bu mod tehlikeli)

## 6. Protected Path Güvenliği

> Bu testleri `--approval none` ile yap ki approval araya girmesin.

- [ ] "read .env.local" → "protected file" hatası
- [ ] "read .bolt-cowork/config.yaml" → reddediliyor
- [ ] "write to .ssh/test" → reddediliyor
- [ ] "copy hello.txt to .ssh/authorized_keys" → reddediliyor
- [ ] "delete .gnupg/something" → reddediliyor
- [ ] "mkdir .ssh/newdir" → reddediliyor
- [ ] "list .ssh" → reddediliyor (dizin root'u da korumalı)

### 6a. Symlink Bypass Denemeleri (Linux/Mac)
- [ ] `ln -s .env.local safe.txt` → "read safe.txt" → reddediliyor
- [ ] `ln -s ../.ssh safe_dir` → "mkdir safe_dir/test" → reddediliyor
- [ ] `ln -s ../.ssh safe_dir` → "copy file.txt to safe_dir" → reddediliyor

## 7. Secret Redaction

- [ ] Config'e bilinen bir API key yaz (ör. `sk-test-12345`)
- [ ] LLM hata verirse → hata mesajında API key `[REDACTED]` olarak görünüyor
- [ ] Step result'ta API key geçiyorsa → `[REDACTED]`
- [ ] Overlapping keys testi: `sk-test` ve `sk-test-abcdef` varsa → `sk-test-abcdef` tam redact ediliyor, parçalanmıyor

## 8. Provider / Fallback Chain

- [ ] Geçerli API key ile → LLM yanıt veriyor
- [ ] Geçersiz API key ile → anlamlı hata, API key redact edilmiş
- [ ] Birden fazla provider tanımlı, ilki başarısız → ikinciye düşüyor (fallback)
- [ ] Tüm provider'lar başarısız → "all providers failed" hatası

## 9. Skill Sistemi

### 9a. Skill Yükleme
- [ ] `./bolt-skills/` içine bir SKILL.md dosyası koy → başlangıçta yükleniyor
- [ ] `~/.bolt-cowork/skills/` içine global skill koy → yükleniyor
- [ ] Aynı isimde skill her iki yerde → project olanı kazanıyor, uyarı loglanıyor
- [ ] Geçersiz YAML frontmatter → uyarı, fallback değerlerle yükleniyor
- [ ] Boş dosya → isim dosya adından türetiliyor, hata yok
- [ ] CRLF satır sonlu SKILL.md → düzgün parse ediliyor

### 9b. Skill Tetikleme
- [ ] `/skill-adı` komutu → ilgili skill aktif oluyor
- [ ] Bilinmeyen skill adı (`/nonexistent-skill`) → "skill not found" hatası
- [ ] auto_trigger: true olan skill → eşleşen komutta otomatik aktif

### 9c. Skill Context Injection
- [ ] Aktif skill varken LLM çağrılınca → system prompt'ta skill içeriği görünüyor (debug/verbose modda kontrol et)

## 10. Context Trimming

- [ ] 20'den fazla mesaj gönder → eski mesajlar kırpılıyor, son 20 kalıyor
- [ ] Çok uzun tek mesaj (>32K karakter) → trimming çalışıyor, crash yok
- [ ] Kırpma sonrası summary mesajı ekleniyor

## 11. Error Handling — Edge Cases

- [ ] LLM boş yanıt dönerse → crash yok, anlamlı hata
- [ ] LLM malformed JSON dönerse → "invalid action" hatası, devam edebiliyor
- [ ] Disk dolu simülasyonu (yazma başarısız) → hata mesajı, crash yok
- [ ] Ctrl+C tam işlem sırasında → graceful shutdown, yarım dosya bırakmıyor

## 12. Hata Mesajı Stili

- [ ] Tüm hata mesajları küçük harfle başlıyor (büyük harf yok)
- [ ] Hata mesajlarının sonunda nokta yok
- [ ] `[DANGEROUS]` etiketinde neden açıklaması var (ör. "permanently removes file")

## 13. Windows Uyumluluğu (Windows'ta test ediyorsan)

- [ ] Banner ASCII-only, kutu karakterleri yok
- [ ] Symlink testleri skip ediliyor (panic yok)
- [ ] Path separator `\` düzgün çalışıyor
- [ ] Renklendirme düzgün görünüyor veya graceful degradation

## 14. Build & Lint Doğrulaması

```bash
go build ./...          # exit code 0
go vet ./...            # temiz
go test ./...           # hepsi PASS
gofmt -l ./...          # boş çıktı (formatlanmamış dosya yok)
go mod tidy && git diff # değişiklik yok
```
