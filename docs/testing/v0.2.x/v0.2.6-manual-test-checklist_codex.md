## 1. CLI Başlangıç

- [x] `bolt-cowork --help` → flagler ve açıklamalar görünüyor
- [!] `bolt-cowork --dir ./test-workspace` → REPL açılıyor, banner (ASCII-only) görünüyor
- [!] `bolt-cowork --dir /nonexistent` → anlamlı hata mesajı, crash yok
- [x] `bolt-cowork --provider nonexistent` → provider bulunamadı hatası
- [x] `bolt-cowork --approval invalid` → geçersiz approval mode hatası

## 2. Config Yükleme

- [!] `~/.bolt-cowork/config.yaml` yokken başlat → varsayılan config ile çalışıyor
- [x] `config.yaml` içinde env var kullan (`${API_KEY}`) → genişletiliyor
- [x] Geçersiz YAML → açıklayıcı parse hatası, panic yok
- [x] Tilde (`~`) path genişletmesi çalışıyor (ör. `skills_dir: ~/my-skills`)

## 3. REPL Temel Komutlar

- [x] Boş input (sadece Enter) → hata yok, yeni prompt
- [x] `/help` → komut listesi görünüyor
- [x] `/dir` → mevcut çalışma dizinini gösteriyor
- [!] `/dir ./subdir` → dizin değişiyor (subdir varsa)
- [x] `/dir /outside/sandbox` → sandbox dışı, reddediliyor
- [x] `/dir -` → önceki dizine dönüyor
- [x] `/quit` veya Ctrl+C → temiz çıkış, "goodbye" mesajı
- [x] Bilinmeyen komut (`/asdf`) → "unknown command" hatası

## 4. Agent Loop — Dosya İşlemleri

- [x] "create a file called hello.txt with content 'merhaba'" → dosya oluşuyor, içerik doğru
- [x] "read hello.txt" → içerik gösteriliyor
- [x] "rename hello.txt to selam.txt" → isim değişiyor
- [x] "copy selam.txt to backup.txt" → kopya oluşuyor
- [x] "delete backup.txt" → onay isteniyor (dangerous), onaylayınca siliniyor
- [x] "list files" → mevcut dosyalar listeleniyor
- [x] "create directory subdir" → klasör oluşuyor

## 5. Approval Gate (Onay Kapısı)

### 5a. Full Approval Mode (`--approval full`)
- [x] Her adımda onay isteniyor (plan + execute)
- [x] Reject yapınca → işlem iptal, dosya değişmemiş
- [x] ApproveAll seçince → kalan adımlar otomatik onaylanıyor

### 5b. Dangerous-Only Mode (`--approval dangerous`)
- [!] read/list gibi güvenli işlemler → otomatik onay, soru yok
- [!] delete/overwrite → onay isteniyor, `[DANGEROUS]` etiketi + neden görünüyor
- [x] ApproveAll seçtikten sonra başka bir dangerous adım gelince → YİNE onay istiyor
- [x] Non-dangerous adımlar ApproveAll sonrası → otomatik geçiyor

### 5c. None Mode (`--approval none`)
- [x] Hiçbir adımda onay istemiyor
- [x] Dangerous işlemler bile direkt çalışıyor (uyarı: bu mod tehlikeli)

## 6. Protected Path Güvenliği

> Bu testleri `--approval none` ile yap ki approval araya girmesin.

- [x] "read .env.local" → "protected file" hatası
- [x] "read .bolt-cowork/config.yaml" → reddediliyor
- [x] "write to .ssh/test" → reddediliyor
- [x] "copy hello.txt to .ssh/authorized_keys" → reddediliyor
- [x] "delete .gnupg/something" → reddediliyor
- [x] "mkdir .ssh/newdir" → reddediliyor
- [x] "list .ssh" → reddediliyor (dizin root'u da korumalı)

### 6a. Symlink Bypass Denemeleri (Linux/Mac)
- [skip] `ln -s .env.local safe.txt` → "read safe.txt" → reddediliyor
- [skip] `ln -s ../.ssh safe_dir` → "mkdir safe_dir/test" → reddediliyor
- [skip] `ln -s ../.ssh safe_dir` → "copy file.txt to safe_dir" → reddediliyor

## 7. Secret Redaction

- [x] Config'e bilinen bir API key yaz (ör. `sk-test-12345`)
- [x] LLM hata verirse → hata mesajında API key `[REDACTED]` olarak görünüyor
- [x] Step result'ta API key geçiyorsa → `[REDACTED]`
- [x] Overlapping keys testi: `sk-test` ve `sk-test-abcdef` varsa → `sk-test-abcdef` tam redact ediliyor, parçalanmıyor

## 8. Provider / Fallback Chain

- [x] Geçerli API key ile → LLM yanıt veriyor
- [x] Geçersiz API key ile → anlamlı hata, API key redact edilmiş
- [x] Birden fazla provider tanımlı, ilki başarısız → ikinciye düşüyor (fallback)
- [x] Tüm provider'lar başarısız → "all providers failed" hatası

## 9. Skill Sistemi

### 9a. Skill Yükleme
- [x] `./bolt-skills/` içine bir SKILL.md dosyası koy → başlangıçta yükleniyor

- [x] Aynı isimde skill her iki yerde → project olanı kazanıyor, uyarı loglanıyor
- [x] Geçersiz YAML frontmatter → uyarı, fallback değerlerle yükleniyor
- [x] Boş dosya → isim dosya adından türetiliyor, hata yok
- [x] CRLF satır sonlu SKILL.md → düzgün parse ediliyor

### 9b. Skill Tetikleme
- [x] `/skill-adı` komutu → ilgili skill aktif oluyor
- [x] Bilinmeyen skill adı (`/nonexistent-skill`) → "skill not found" hatası
- [x] auto_trigger: true olan skill → eşleşen komutta otomatik aktif

### 9c. Skill Context Injection
- [x] Aktif skill varken LLM çağrılınca → system prompt'ta skill içeriği görünüyor (debug/verbose modda kontrol et)

## 10. Context Trimming

- [x] 20'den fazla mesaj gönder → eski mesajlar kırpılıyor, son 20 kalıyor
- [x] Çok uzun tek mesaj (>32K karakter) → trimming çalışıyor, crash yok

## 11. Error Handling — Edge Cases

- [x] LLM boş yanıt dönerse → crash yok, anlamlı hata

- [!] Hata mesajlarının sonunda nokta yok
