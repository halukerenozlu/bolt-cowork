    • os.UserHomeDir() kullanımı runtime config/init/skill lookup için var; testlerde çoğunlukla t.TempDir() ve t.Setenv kullanılmış. Windows'ta os.UserHomeDir() davranışı nedeniyle internal/skill/loader_test.go içinde global scope testi fallback kabul ediyor.
    • ShellCheck çalıştırılamadı: bu ortamda shellcheck komutu kurulu değil.
    • README/CHANGELOG/spec dokümantasyonunda bazı sürüm/roadmap ifadeleri v0.2.6, v0.3 "next" ve dangerous-only isimleri açısından manuel son göz ister.
    • Piped stdin ile REPL komutlarını otomatik doğrulama Windows/readline nedeniyle canlı REPL davranışını temsil etmedi; bu maddeleri birlikte/manual bıraktım.

    Gerçek API ile Yapılacaklar

    Bu maddeler başarılı LLM planı veya execution aşamasına geçiş gerektiriyor. Sahte API ile yalnızca auth failure görülüyor, bu yüzden gerçek provider key olmadan güvenilir doğrulanmadı.

    • "rename hello.txt to selam.txt" -> isim değişiyor
    • "copy selam.txt to backup.txt" -> kopya oluşuyor
    • Geçerli API key ile -> LLM yanıt veriyor
    • [DANGEROUS] etiketinde neden açıklaması var
    • Bir dosya yazma (write) işlemi sırasında Ctrl+C basılırsa yarım/bozuk dosya kalıyor mu
    • Kullanıcı planı reddedip geri bildirim verdiğinde LLM'e giden yeni prompt'ta önceki plan ve kullanıcı yorumu yer alıyor mu
    • Plan revizyonu sırasında /dir veya /model gibi komutlar süreci bozuyor mu
    • LLM'e gönderilen system prompt içinde config secret'ları sızıyor mu
    • Spinner aktifken pencere boyutu değişince çıktı bozuluyor mu; daha anlamlı test için uzun LLM çağrısı gerekiyor

    Manuel / Sistemsel Edge-Case Maddeler

    Bu maddeler gerçek API değil, OS veya özel test ortamı/senaryosu gerektiriyor.

    • Disk dolu simülasyonu -> hata mesajı, crash yok
    • allowed_dirs içindeki dizin silinince agent davranışı

    Tamamlanan/sonuçlanan edge-case notları:

    • Windows reserved names -> FAIL, CON, PRN, AUX oluşabildi; özel Bolt koruması yok görünüyor
    • SkillScope enum dışı değer -> PASS, SkillScope.String() default olarak unknown döndürüyor
    • Boşluklu path tırnak parsing -> FAIL, slash command parser strings.Fields kullandığı için quoted args desteklemiyor
    • allowed_dirs silinmesi -> PARTIAL, sandbox layer nonexistent root için hata veriyor; ancak CLI başlangıçta nonexistent --dir ile REPL açma bug'ı ayrıca F-001 olarak kayıtlı

    Blocked / Partial Kapanış Notları

    • Plan revizyonu sırasında /dir veya /model testi -> PARTIAL/BLOCKED. Canlı binary'de plan revision feedback prompt'u görünmediği için revizyon sırasında slash command davranışı güvenilir test edilemedi. Bu madde F-012 çözülmeden anlamlı şekilde kapanamaz.
    • LLM system prompt secret sızıntısı -> PARTIAL. Redaction testleri ve config secret redaction kapsamı var, ancak debug/verbose payload görüntüleme modu olmadığı için gerçek provider'a giden system prompt canlı olarak doğrulanmadı.
    • Spinner/resize görsel testi -> PARTIAL. Terminal daraltıldığında banner ciddi şekilde kırpıldı/sardı, ancak approval ekranları, sonuç metni ve prompt okunabilir kaldı; ham ANSI escape karakteri görülmedi. Spinner aktif anı çok kısa sürdüğü için tam "spinner aktifken resize" doğrulanamadı.
    • Write sırasında Ctrl+C atomicity testi -> PARTIAL. atomic-ctrlc.txt önce original-content ile hazırlandı; overwrite komutu execute approval sonrası çok hızlı tamamlandı ve Ctrl+C result approval aşamasında yakalandı. Dosya tam replacement-content içeriyor; yarım/bozuk dosya yok, fakat gerçek write anı yakalanamadı.
